Advanced usage

Refer to another note

You can refer to another with the syntax of:


[{label}](marknote://{title})

Where {label} is the text you want to display and {title} is the title of the note you want to refer to.
Note, you need to use %20 to replace all spaces in the title.

Use tag to organize your notes

Tags is a good way to keep your notes orgainized.
To add a tag to your note, just add a line under the title of your note, in the format of
Tags:{tag1},{tag2}One note can be associated with multiple tags.

Use Marknotes to build static blog website

Do you believe my blog site is completed done with this app?
By following the steps below you can also build your own!

1. Configure
The 1st thing is to make some quick configuration.
In Search box, type in settings, you will find a note with title settings. Click and select that note.

Item Description
SiteName The name of your site
Base URL The root URL of your site
SiteDesc The description of your site, will show at the bottom of each page
github-user your github account, optional
twitter-user Your account name on twitter, optional

2. Filter the notes to export
By default, MarkNotes will exports all notes into static site. If you don't want that, you can filter your notes by providing values to Search box and/or tag dropdown.

3. Export
Now you can export the selected notes with menu item: File-> Export all notes as blog site
All your notes will be exported into a folder you specified, and index files and sitemap are all generated.
You can simply deploy the folder to any server that provides HTTP service and make your site work.

FAQ

1. Why I cannot refer to images with http url in my notes? why my http:// link does not work?

Refer to Apple's document , after iOS 9 and OS X v10.11, App Transport Security (ATS) enforces best practices in the secure connections between an app and its back end. ATS only allows safe HTTPS requests.
With this app, you can refer to HTTPS resources or links. HTTP ones will fail.
